Settings

Google Sheets, Google Sign-In, Encryption, and API Key

📊 Google Sheets Setup
Required for multi-user data sharingWithout this, each user's data stays on their own device. Follow the one-time setup to connect a shared Google Sheet.
  1. Open Google Sheets → create a new blank spreadsheet
  2. Click Extensions → Apps Script
  3. Delete all code → paste the contents of AppScript.gs (same folder as this app)
  4. Click Save, then Deploy → New Deployment
  5. Type: Web App | Execute as: Me | Access: Anyone
  6. Click Deploy → Authorize → Allow
  7. Copy the Web App URL and paste below
🔵 Google Sign-In Setup
⚠️ HTTP server requiredGoogle Sign-In will NOT work if you open index.html directly as a file. Use start.bat (double-click it) to launch a local server, then access the app at http://localhost:8080.
  1. Go to console.cloud.google.com → create or select a project
  2. Go to APIs & Services → Credentials
  3. Click Create Credentials → OAuth 2.0 Client IDs
  4. Application type: Web application
  5. Under Authorised JavaScript origins add: http://localhost:8080
  6. Click Create and copy the Client ID
  7. Paste the Client ID below and click Save
🔐 Security — Decryption Key
How encryption worksEach OI officer enters an encryption key in their Settings. All text details in their reports are encrypted (AES-256) before being saved to Google Sheets. The Admin enters the same key here to decrypt and view the full details. Charts and status fields are always visible without the key.

🔐 Encryption Key
How it worksEnter the encryption key provided to you by the Administrator. All sensitive text in your reports (MoU details, visit details, engagement notes) will be encrypted with this key before being stored on Google Sheets. The Admin holds the same key and uses it to decrypt your submissions.
⚠️ ImportantThis key is your responsibility. Do not share it digitally — it should be handed over in person. If you lose the key or change it, the Admin will not be able to decrypt your previous reports.
